# Product

## Register

product

## Users

Individual Linux desktop users who want a personal screen companion and occasionally adjust how it looks or responds without using command-line flags or environment variables.

## Product Purpose

Pixel Pet provides a friendly visual controller for choosing, configuring, previewing, pausing, and positioning a persistent desktop companion. Success means users can understand the pet's current behavior, change it with immediate feedback, and then close the controller without stopping the companion.

## Brand Personality

Playful, calm, unobtrusive. The product should feel like caring for a tiny companion while remaining as predictable and efficient as native desktop preferences.

## Anti-references

Avoid dense developer dashboards, childish toy-store styling, neon gamer aesthetics, generic nested card grids, and decorative animation that competes with the pet preview.

## Design Principles

1. Companion first: keep the pet visible and emotionally legible while configuring it.
2. Familiar controls: use standard desktop settings patterns and plain language.
3. Live feedback: reflect safe changes immediately and visibly.
4. Quiet defaults: preserve focus and make optional reactions easy to disable.
5. Reversible control: destructive or disruptive actions must be explicit; ordinary settings should be easy to restore.

## Accessibility & Inclusion

Target WCAG AA. Support complete keyboard navigation, visible focus, screen-reader labels, and state communication that never relies on color alone. Respect reduced-motion preferences by pausing nonessential preview animation.
